# SKDO

СКДО — система консолидированных данных обследования и мониторинга, которая собирает,
проверяет и подготавливает инженерные данные для последующей идентификации скрытого
состояния.

## Что делает система

- хранит список объектов и дерево элементов;
- хранит дефекты, измерения, параметры среды, вмешательства и испытания;
- считает полноту данных и готовность к следующему расчётному шагу;
- формирует и выгружает `observation_package`;
- даёт два интерфейса: API для интеграций и простой web-интерфейс для инженера.

## Что не делает версия 1.0

- конечно-элементный расчёт;
- идентификацию скрытых параметров;
- прогноз остаточного ресурса;
- рекомендации по ремонту.

## Быстрый старт

### Вариант 1. Запуск через Docker Compose

```bash
docker compose up --build
```

После запуска будут доступны:

- инженерный интерфейс: `http://localhost:8501`
- API: `http://localhost:8000`
- Swagger/OpenAPI: `http://localhost:8000/docs`

### Вариант 2. Локальный запуск без Docker

Требуется Python `3.11+`.

```bash
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -e ".[dev,ui]"
alembic upgrade head
python -m uvicorn apps.api.main:app --host 127.0.0.1 --port 8000
```

Во втором окне терминала:

```bash
python -m streamlit run apps/ui/app.py --server.address 127.0.0.1 --server.port 8501
```

## Как запускать backend и UI отдельно

### Backend

```bash
python -m uvicorn apps.api.main:app --host 127.0.0.1 --port 8000
```

### UI

```bash
python -m streamlit run apps/ui/app.py --server.address 127.0.0.1 --server.port 8501
```

### Адреса в браузере

- UI: `http://127.0.0.1:8501`
- API: `http://127.0.0.1:8000`
- Swagger: `http://127.0.0.1:8000/docs`

## Очень простая инструкция для инженера

### 1. Как открыть систему

- Откройте в браузере `http://127.0.0.1:8501`.
- Слева, в поле `Адрес backend API`, оставьте `http://127.0.0.1:8000`.
- Нажмите кнопку `Проверить связь`.

Если всё в порядке, появится сообщение `Связь с API есть`.

### 2. Как создать объект

- На главной странице справа выберите `Создать объект`.
- Заполните хотя бы поля `Код объекта` и `Название объекта`.
- Нажмите `Сохранить`.

После этого объект появится в списке, а слева его можно выбрать как текущий.

### 3. Как добавить элементы

- Откройте вкладку `Элементы`.
- Заполните форму `Добавить элемент`.
- Нажмите `Сохранить`.

### 4. Как добавить дефекты

- Откройте вкладку `Дефекты`.
- Заполните форму `Добавить дефект`.
- Нажмите `Сохранить`.

### 5. Как загрузить измерения из CSV/XLSX

- Откройте вкладку `Измерения`.
- В блоке `Канал измерений` создайте канал и нажмите `Сохранить`.
- Ниже выберите этот канал.
- Нажмите `Browse files` и выберите CSV или XLSX.
- Проверьте предпросмотр.
- Нажмите `Загрузить измерения`.

Минимально в файле должны быть колонки:

- `timestamp` или `дата`
- `value` или `значение`

Пример файла лежит здесь: `sample_data/measurements_template.csv`

### 6. Как посмотреть оценку данных

- Откройте вкладку `Оценка данных`.
- Система покажет общую оценку и список того, чего не хватает.

### 7. Как посмотреть готовность

- Откройте вкладку `Готовность`.
- Там видно, готов ли набор данных для следующего шага.

### 8. Как выгрузить пакет

- Откройте вкладку `Пакет`.
- Нажмите кнопку `Сформировать пакет`.
- Затем нажмите `Скачать observation_package.json`.

## Структура репозитория

- `apps/api/` — FastAPI-приложение и роуты.
- `apps/core/` — схемы, сервисы, импорт/экспорт, расчёт индексов.
- `apps/db/` — SQLAlchemy-модели и сессия.
- `apps/ui/` — простой инженерный интерфейс на Streamlit.
- `alembic/` — миграции.
- `docs/` — доменная документация и API-контракт.
- `sample_data/` — demo dataset и пример файла измерений.
- `tests/` — unit и integration tests.

## Основные выходы

- `element_state_observation_record`
- `information_sufficiency_index`
- `identification_readiness_report`
- `critical_missing_data_list`
- `observation_package`

## Файлы интерфейса

- `apps/ui/app.py` — основной экран интерфейса.
- `apps/ui/api_client.py` — обращения к backend API.
- `apps/ui/import_utils.py` — разбор CSV/XLSX перед загрузкой.
- `sample_data/measurements_template.csv` — пример файла измерений.
