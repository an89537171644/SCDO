# SKDO

СКДО - система консолидированных данных обследования и мониторинга.

Проект собирает, нормализует, проверяет и экспортирует инженерные данные по объектам, элементам, дефектам, измерениям, среде, вмешательствам и испытаниям. Версия `v1.1` расширяет MVP до слоя `Mechanics-ready data layer`, пригодного для следующего шага механической интерпретации и идентификации.

## Что умеет система

- хранить объекты и дерево элементов;
- хранить дефекты, измерения, среду, ремонты, испытания, медиа и качество данных;
- считать `information_sufficiency_index`;
- считать `identification_readiness_report`;
- формировать `observation_package`;
- давать два интерфейса:
  - FastAPI + Swagger для интеграций;
  - Streamlit UI для инженера без ручных API-запросов.

## Что добавлено в v1.1

- механические поля элемента: сечения, инерции, моменты сопротивления, расчетные длины, направленные жесткости опирания;
- материалоспецифичные поля для стали и железобетона;
- усиленная модель дефекта: механизм, тяжесть, зона, локальная координата, скорость роста, метод выявления;
- метаданные канала и измерения: ось, знак, сценарий нагрузки, температурная компенсация, агрегация, прибор, калибровка;
- расширенный `observation_package` с `section_properties` и `critical_missing_data_by_element`;
- покрытийно-взвешенный `information_sufficiency_index`;
- расширенный `identification_readiness_report` по механическим классам;
- инженерный UI с шаблонами элементов, вкладкой медиа и статусом заполненности по важным элементам.

## Быстрый старт

### Вариант 1. Docker Compose

```bash
docker compose up --build
```

После запуска будут доступны:

- UI: `http://127.0.0.1:8501`
- API: `http://127.0.0.1:8000`
- Swagger: `http://127.0.0.1:8000/docs`

### Вариант 2. Локальный запуск

Нужен Python `3.11+`.

```bash
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -e ".[dev,ui]"
alembic upgrade head
python -m uvicorn apps.api.main:app --host 127.0.0.1 --port 8000
```

Во втором окне терминала:

```bash
python -m streamlit run apps/ui/app.py --server.address 127.0.0.1 --server.port 8501
```

## Очень простой сценарий для инженера

### 1. Открыть систему

- откройте `http://127.0.0.1:8501`;
- слева оставьте адрес API `http://127.0.0.1:8000`;
- нажмите `Проверить связь`.

### 2. Создать объект

- на главной странице справа выберите `Создать объект`;
- заполните `Код объекта` и `Название объекта`;
- нажмите `Сохранить`.

### 3. Добавить элемент

- откройте вкладку `Элементы`;
- выберите шаблон, например `Стальная балка` или `Ж/б балка`;
- заполните название и основные поля;
- нажмите `Сохранить`.

### 4. Добавить дефект

- откройте вкладку `Дефекты`;
- выберите элемент;
- заполните тип, место и основные параметры;
- нажмите `Сохранить`.

### 5. Загрузить измерения

- откройте вкладку `Измерения`;
- сначала создайте канал и нажмите `Сохранить`;
- затем выберите файл CSV/XLSX;
- проверьте предпросмотр;
- нажмите `Загрузить измерения`.

### 6. Посмотреть полноту и готовность

- вкладка `Оценка данных` показывает полноту;
- вкладка `Готовность` показывает, чего хватает для следующего шага расчета.

### 7. Загрузить медиа

- откройте вкладку `Медиа`;
- выберите файл и элемент;
- нажмите `Сохранить`.

### 8. Выгрузить пакет

- откройте вкладку `Пакет`;
- нажмите `Сформировать пакет`;
- нажмите `Скачать observation_package.json`.

## Структура репозитория

- `apps/api/` - FastAPI приложение и маршруты
- `apps/core/` - схемы, сервисы, импорт, экспорт, индексы качества
- `apps/db/` - SQLAlchemy модели и сессии
- `apps/ui/` - Streamlit интерфейс инженера
- `alembic/` - миграции
- `docs/` - доменные и контрактные документы
- `sample_data/` - demo dataset и шаблоны импорта
- `tests/` - unit и integration tests
