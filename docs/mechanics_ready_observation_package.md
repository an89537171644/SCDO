# Mechanics-Ready Observation Package

Версия `v1.1` расширяет `observation_package` так, чтобы пакет был пригоден для следующего шага механической интерпретации и идентификации.

## Что добавлено

В `element_state_observation_record` теперь обязательно формируются блоки:

- `actual_material`
- `boundary_conditions`
- `section_properties`
- `data_coverage`
- `critical_missing_data_by_element`

## Назначение блоков

`actual_material`
- фактическая марка материала;
- фактические механические характеристики;
- RC/steel-специфичные поля вроде `rebar_class`, `steel_grade_actual`, `corrosion_loss_mm`.

`boundary_conditions`
- тип опирания;
- суммарная жесткость;
- направленные коэффициенты `support_k*`, `support_r*`;
- податливость узлов `joint_flexibility*`.

`section_properties`
- имя и семейство сечения;
- инерции;
- моменты сопротивления;
- крутильная постоянная;
- расчетные длины потери устойчивости.

`data_coverage`
- временное покрытие;
- пространственное покрытие;
- плотность наблюдений;
- уровень неопределенности.

`critical_missing_data_by_element`
- список недостающих данных именно по данному элементу;
- используется в UI для подсветки инженерных пробелов.

## Совместимость

Пакет остается обратно совместимым:

- версия экспорта остается `v1.1`;
- старые поля не удаляются;
- новые поля добавлены только расширением контракта.
